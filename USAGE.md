# 股票复盘助手使用说明

这是一个个人自用的股票复盘与辅助决策系统。它不会自动交易，也不会直接输出“买入 / 卖出”指令；当前版本重点是帮你优先发现暴雷风险、K线风险、仓位风险和行业集中风险。

## 1. 启动应用

```bash
cd stock-review-assistant
source .venv/bin/activate
streamlit run app.py
```

浏览器打开：

```text
http://127.0.0.1:8501
```

首次打开时，如果系统里还没有账号，会先要求创建管理员账号。创建后再登录使用。

如果要在手机或微信里看，需要让手机和电脑在同一网络下，并用电脑局域网 IP 访问，例如：

```text
http://你的电脑IP:8501
```

如果手机不在同一网络下，需要公网地址，例如：

- 部署到 Streamlit Cloud
- 使用 Cloudflare Tunnel
- 使用 ngrok
- 部署到自己的云服务器

拿到公网地址后，打开“系统设置”里的“手机访问二维码”，填入公网 URL，即可生成二维码。二维码只负责打开地址，不能把本地应用自动变成公网应用。

## 2. 登录系统

当前版本已经加入登录保护和用户数据隔离：

- 第一次使用时创建管理员账号
- 之后所有页面都需要登录
- 系统设置里可以创建朋友账号
- 侧边栏和系统设置都可以退出登录
- 每个账号只看到自己的持仓、报告和交易日志
- 每个账号独立配置 DeepSeek API Key、微信推送和自动推送

从旧版升级时，原有持仓、报告和配置会自动归入数据库中最早创建的账号，不需要重新录入。

## 3. 第一次使用流程

1. 打开“持仓管理”
2. 新增持仓
3. 填写股票代码、股票名称、买入价格、持仓数量、买入日期
4. 补充所属行业、投资逻辑
5. 如果这只股票需要重点关注，勾选“重点监控”
6. 回到首页查看暴雷风险和仓位风险

如果股票较多，可以打开“持仓管理”的“批量导入”标签：

1. 下载 Excel 模板或 CSV 模板
2. 按模板填写股票代码、股票名称、买入价格、持仓数量、买入日期
3. 可选填写所属市场、所属行业、投资逻辑、重点监控和备注
4. 上传填写好的表格
5. 检查页面预览
6. 点击“确认导入这些持仓”

导入会逐行处理。某一行格式错误时，页面会显示具体行号和错误原因，其他正确行仍可正常导入。

## 4. 首页怎么看

首页现在是风险优先视图，并且默认读取上次刷新结果，不会每次打开都自动拉行情。

重点看这几块：

- 暴雷风险提醒：第一优先级
- 今日风险等级：正常 / 注意 / 风险预警 / 严重风险
- 风险明细：每条风险都会说明原因和依据
- 行业集中度：某个行业仓位超过 40% 会提醒
- 每日盈亏简表：查看当前浮动盈亏和收益率

第一次打开如果显示“暂无缓存结果”，点击：

```text
刷新风险数据
```

刷新后结果会保存下来。以后打开首页会直接显示上次刷新结果，速度更快。需要最新数据时再手动刷新。

首页把刷新参数收在“更新数据与手动价格”中，打开页面时先显示结果，不会阻塞历史数据。

风险等级含义：

- 正常：暂未发现明显风险
- 注意：有需要留意的信号
- 风险预警：风险较明显，需要谨慎观察和控制仓位
- 严重风险：优先处理，避免情绪化操作

## 5. 暴雷风险规则

当前版本默认规则：

财报 / 公告文本命中以下关键词会提醒：

- 亏损
- 净利润下降 / 下滑
- 业绩下降 / 下滑
- 营收下降
- 现金流恶化
- 商誉减值
- 监管问询
- 立案调查
- ST
- 退市风险
- 减持
- 质押
- 债务逾期
- 重大诉讼

K线风险包括：

- 放量下跌
- 跌破 MA20
- 跌破 MA60
- 连续 3 天下跌

仓位风险包括：

- 单只股票仓位超过 30%
- 总股票仓位超过 70%，需要你在首页填写账户总资产
- 某个行业仓位超过 40%

### 暴雷风险中心

打开“暴雷风险中心”页面，可以集中查看：

- 仓位风险
- 行业集中度
- K线风险
- 行情获取失败或数据不足提示
- 财报 / 公告 / 新闻文本关键词扫描

“持仓风险扫描”会按规则生成风险明细和行业集中度。“财报/公告文本扫描”不调用 AI，只做本地关键词扫描，适合快速判断公告里是否出现亏损、监管问询、减持、质押、退市风险等明显风险词。

行情扫描现在按以下顺序工作：

1. 腾讯历史日 K
2. 新浪历史日 K
3. 东方财富历史日 K
4. 最近一次本地缓存

多只持仓会并发获取，同一只股票的价格和 K 线风险共用一次结果。页面会显示有效行情数量、是否使用缓存、数据更新时间和具体价格来源。单只股票失败不会中断其他持仓。

## 6. 财报公告分析

打开“财报公告分析”页面：

1. 输入标题
2. 粘贴公告、财报或新闻文本
3. 点击“生成摘要”

系统会先做暴雷关键词扫描，然后调用 DeepSeek 生成通俗摘要。

注意：如果原文信息不足，系统应提示“数据不足，不做判断”。不要只凭一段文本做交易决策。

### 每日消息面汇总

“财报公告分析”页面还提供“每日消息汇总”：

1. 系统会读取你的持仓、所属行业、投资逻辑和重点监控标记
2. 你可以填写关注领域，例如新能源、银行、白酒、AI算力等
3. 点击“自动抓取持仓新闻 / 公告”
4. 也可以手动补充粘贴新闻、公告、财报材料
5. 点击“生成每日消息汇总”

报告会按你的持仓和关注领域输出：

- 今日最重要的消息
- 对持仓的影响
- 行业/关注领域变化
- 暴雷风险扫描
- 明日观察清单
- 推荐关注/优先观察

第一版不会自动编造新闻。如果当天没有粘贴材料，系统会说明数据不足，并给出应该关注哪些信息。

自动抓取使用免费公开源，可能遇到接口超时、SSL、反爬或数据为空。失败时页面会显示失败原因，你可以手动补充材料。

## 7. 微信推送配置

打开“系统设置”页面，找到“微信推送配置”。

支持：

- Server酱
- PushPlus
- 暂不确定

填写：

- 推送类型
- SendKey 或 PushPlus Token
- 勾选“启用微信推送”
- 点击“保存推送配置”

可以点击“发送测试推送”验证是否可用。

首页的“推送风险摘要到微信”会推送：

- 暴雷风险
- K线风险
- 仓位风险
- 数据不足提示

### 自动推送

打开“系统设置”页面，找到“自动推送配置”：

1. 勾选“启用自动推送任务”
2. 填写计划推送时间，例如 `09:00`
3. 选择是否获取当前行情
4. 选择是否检查 K线风险
5. 可选勾选“同时生成每日复盘报告”
6. 点击“保存自动推送配置”

页面里的“立即执行一次自动推送”可以马上测试风险摘要推送。

真正每天自动执行，需要本机定时任务调用脚本：

```bash
cd 'stock-review-assistant'
source .venv/bin/activate
python scripts/run_auto_push.py
```

如果只是临时测试一次，即使没有启用自动推送，也可以运行：

```bash
python scripts/run_auto_push.py --force
```

注意：本地自动推送要求电脑开机、网络可用，并且 Streamlit 数据库里的微信推送配置有效。

## 8. DeepSeek 配置

普通用户可以直接在页面里配置：

1. 打开“系统设置”
2. 在“DeepSeek 配置状态”下填写 API Key
3. 模型选择 `deepseek-v4-pro`
4. AI 请求超时秒数建议设置为 90 秒
5. 失败自动重试次数建议设置为 1 次
6. 点击“保存 DeepSeek 配置”
7. 点击“测试 DeepSeek 连接”

配置会保存到本地 SQLite 数据库中。用户不需要打开终端编辑 `.env`。

如果你习惯用环境变量，也可以在 `.env` 中配置：

```bash
DEEPSEEK_API_KEY=你的key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-pro
DEEPSEEK_THINKING=enabled
DEEPSEEK_REASONING_EFFORT=high
```

打开“系统设置”页面可以测试 DeepSeek 连接。

如果遇到 `Read timed out`：

- 先稍后重试
- 把 AI 请求超时秒数调到 90-120 秒
- 失败自动重试次数设置为 1
- 报告内容较多时，可以临时关闭 K线摘要，或把模型切换为 `deepseek-chat`

## 9. 每日复盘报告

打开“每日复盘报告”页面后，系统会先显示历史复盘，不会立刻拉行情或生成新报告。

你可以：

- 直接查看最新历史报告
- 展开更早的历史报告
- 在“向 DeepSeek 追问”里基于最新报告提问题
- 点击“生成并保存每日复盘”生成新报告

复盘风格已设置为严格、克制、专业：

- 不祝贺
- 不幼稚类比
- 不重复免责声明
- 每个结论必须有依据
- 数据不足时直接说明不能判断

### 定时生成报告

项目提供了命令行脚本：

```bash
cd stock-review-assistant
source .venv/bin/activate
python scripts/generate_daily_review.py
```

这个脚本会生成一份“每日自动复盘报告”，并保存到历史复盘中。

如果你想每天固定时间自动生成，可以把上面的命令加入 macOS `launchd` 或 `cron`。注意：电脑必须开机，且网络、DeepSeek API、行情接口可用。

## 10. 免费数据源说明

当前版本通过 akshare 使用腾讯、新浪和东方财富三个 A股日线来源。免费数据源仍可能出现失败、延迟或字段变化。

如果行情失败：

- 首页可手动输入当前价格
- 如果以前成功获取过，会自动使用最近行情或 K 线缓存
- 缓存数据会明确显示采集时间和原始来源
- 没有缓存时，系统会提示“数据不足，不做判断”
- 不会强行给出结论

打开“系统设置”最下方的“行情缓存”，可以查看股票代码、缓存类型、价格、原始来源和采集时间。

行情请求默认 8 秒超时，可在 `.env` 中调整：

```bash
MARKET_TIMEOUT_SECONDS=8
```

## 11. 单只股票 K线图

打开“单只股票分析”页面：

1. 选择持仓或手动输入股票代码
2. 选择市场和 K线天数
3. 选择要显示的均线，例如 MA5 / MA10 / MA20 / MA60
4. 点击“开始分析”

图表现在包含：

- 蜡烛 K线：红色代表上涨，绿色代表下跌
- 均线：用于观察不同周期趋势
- 成交量：用于观察是否放量上涨或放量下跌
- MACD：用于观察趋势动能变化
- RSI：用于观察短线过热或超跌

鼠标悬停在图上可以查看同一天的价格、成交量和指标。技术图只用于辅助观察，不构成买入或卖出建议。

## 12. 交易日志

打开侧栏“交易日志”，可以记录：

- 关联持仓或手动填写股票
- 买入、卖出、加仓、减仓、观察或取消计划
- 当时价格和数量
- 做出判断的事实依据
- 当时情绪
- 是否遵守纪律及下次改进点

交易日志只记录你已经做过或计划观察的行为，不连接券商，也不会执行真实交易。页面支持查看、筛选、修改和删除记录，并统计日志数、实际交易记录数、记录成交额和冲动记录。

## 13. 使用纪律

本系统适合做复盘和风险提醒，不适合替你做交易决定。

看到风险预警时，建议先问自己：

- 风险来自财报、K线还是仓位？
- 依据是否充分？
- 是否只是短期波动？
- 我的仓位是否过重？
- 如果继续下跌，我能承受多少？

系统可以帮你更清醒，但最终决策仍由你负责。
